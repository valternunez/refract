# Token footprint

> Generated by `pnpm bench` (`benchmarks/tokens.mjs`). Re-run to refresh.

**For 3 viewport(s) of the demo-site: a render_responsive response costs ~4,239 tokens (downscaled, Claude) vs ~17,667 full-res — 76% smaller. The findings JSON alone is ~623.**

Page: `examples/demo-site/index.html` · viewports: `mobile, tablet, desktop`

Tokens an agent pays for one `render_responsive` response, by response shape and model:

| mode | text | claude | gpt-4o | gemini |
|---|---|---|---|---|
| findings-only | 623 | 623 | 623 | 623 |
| downscaled | 643 | 4,239 | 3,958 | 3,739 |
| full-res | 643 | 17,667 | 4,298 | 8,899 |

- **findings-only** — manifest + findings JSON, no images (text only).
- **downscaled** — what Refract returns: previews capped at 800px wide. *(default)*
- **full-res** — what a naive screenshot tool would dump.

Per-viewport image dimensions (full-resolution vs downscaled preview):

| preset | full-res | preview |
|---|---|---|
| mobile | 1206×2622 | 800×1739 |
| tablet | 1668×2420 | 800×1161 |
| desktop | 3024×1964 | 800×520 |

### How it's counted

Text is tokenized with `gpt-tokenizer` (o200k_base); it's a proxy for Claude/Gemini
text (within ~10–20%), which is fine because images dominate the total. Base64 image
data is **not** counted as text — the MCP sends images as image content blocks, so they
cost *image* tokens per each model's published formula:

- **Claude:** `ceil(w/28) * ceil(h/28)` — [docs](https://platform.claude.com/docs/en/build-with-claude/vision)
- **GPT-4o (high detail):** fit to 2048², scale shortest side to 768, then `85 + 170 * tiles` (512px tiles) — [docs](https://platform.openai.com/docs/guides/images)
- **Gemini 2.0:** `258` if ≤384², else `ceil(w/768) * ceil(h/768) * 258` — [docs](https://ai.google.dev/gemini-api/docs/tokens)

These are estimates, not billed counts; they exist to show the *relative* cost of
Refract's findings-first, downscaled responses versus dumping full-resolution images.
